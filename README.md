# 概要
Djangoの勉強で作ったWebアプリ。
<br>
<br>
ユーザーがアップロードした画像から、Happy Tree Friendsというアニメの登場人物であるカドルス、ギグルス、ランピーの３人のキャラクターの中からどのキャラクターが画像に映っているか判別してくれます。
<br>
## 識別モデルの作成について
それぞれのキャラクターの画像ファイルが35枚とかなり少なかったため、VGG16の転移学習でモデル作成を行い、最終的にはそれぞれのキャラクターの画像ファイルが35枚とかなり少なかったため、VGG16の転移学習でモデル作成を行い、最終的にはvalidation dataで90%前後の精度を出すことができました。
<br>
　ただ、アニメの画像を扱っているので転移学習を行わずに簡単なモデルを作っても良かった気もしました。しかし、今回はDjangoの勉強をメインにしていたので、モデル作成にそこまで時間をかけることはしませんでした。
<br>
## 問題点
システム的な問題はないのですが、画像判定で出来ないことがいくつかあるので、それを記述しておきます。
<br>
<br>
<br>
<b>・同一画像内に複数のキャラクターがいる場合にそれぞれのキャラクターを判別できない。</b><br>
<br>
<b>・学習したキャラクター以外のキャラクターを「others」などのように判別できず、学習した3人のうちのどれかで判別してしまう。</b><br>

## 最後に
Djangoは完全に初学ではなかったので、知識の確認みたいな感じになったけれど、webの仕組みを確認する良い機会になった。キャラクターの判別の面に関しては完全に手抜き状態だったのでいつかちゃんとしたものを作りたいです。また、2ヶ月くらい前に作成したものについて書いているので、もしかしたら記述抜けがあるかもしれません。
